# 使用教程

本文说明仓库中 **三个** Python 工具的用途、依赖与命令行用法：

| 脚本 | 作用 | 平台 |
|------|------|------|
| `arm_trajectory_transform.py` | 手臂轨迹：FK → 施加 **H** → **迭代 IK**（阻尼最小二乘至收敛）→ 写回关节列 | x86 / aarch64 |
| `arm_trajectory_transform_jacobian.py` | 同上 FK 与 **H**，用 **固定步数雅可比增量**（默认 **2 步**：先在原始 `track_joint` 上算 Δq₁，再在叠加后的位形上算 Δq₂）→ 写回 | x86 / aarch64 |
| `mujoco_trajectory_sim.py` | 同上读模型与 `.data`；可选 **`--tf-arm`** 在回放时在线做与 IK 变换脚本相同的 SE(3)+迭代 IK（含 **`both`** 双臂） | **仅 x86** |

> **平台说明**：MuJoCo 仿真（`mujoco_trajectory_sim.py`）**仅在 x86_64 环境中可用**，Orin / aarch64 不支持。变换脚本（`arm_trajectory_transform*.py`）基于 Pinocchio，x86 与 aarch64 均可运行。

**选用提示**：要末端尽量贴合 `H ⊗ FK`，优先 **`arm_trajectory_transform.py`**；要显式「在原始关节角附近做一两步线性修正」或更省算力，可用 **`arm_trajectory_transform_jacobian.py`**（大位姿变化时精度通常弱于迭代 IK）。

**工程文件**：除上表三个入口脚本外，**`arm_transform_common.py`** 为左右臂偏移与 `ArmPassSpec` 的公共解析逻辑（由两个 `arm_trajectory_transform*.py` 与仿真里的 **`--tf-*`** 间接使用）。

---

## 环境依赖

- Python 3
- **Pinocchio**（`ros-humble-pinocchio`）— x86 与 aarch64 均需
- **NumPy**
- **SciPy**（两个 `arm_trajectory_transform*.py` 均使用 `scipy.spatial.transform.Rotation`）
- **MuJoCo**（Python 包 `mujoco`）— **仅 x86_64**；aarch64 不需安装

两个变换脚本共用 **`arm_transform_common.py`** 解析 **`--arm both`** 与 **`--offset-left` / `--offset-right`** 等。

在 x86 环境运行 `mujoco_trajectory_sim.py` 时，请保证 MJCF 内 `meshdir` 等相对路径能解析（通常从仓库根目录运行）。

## 一、`arm_trajectory_transform.py`

### 1.1 功能概要

1. 读取轨迹文件：首行为关节列名，其后每帧一行数值（逗号分隔，与 `read_trajectory_table` 一致；支持行末尾随逗号产生的空列）。
2. 用 Pinocchio 模型将**整行**中能在模型里找到名字的关节写入 `qpos`（其余列忽略），做正运动学得到末端 `T_ee`。
3. 按 `x,y,z,rx,ry,rz` 构造齐次变换 **H**（平移 + 固定轴欧拉旋转），计算目标位姿：
   - `--compose left`（默认）：`T' = H @ T_ee`
   - `--compose right`：`T' = T_ee @ H`
4. **阻尼最小二乘 IK**（多步迭代直至 `--ik-tol` 或 `--ik-iters`），更新指定关节，再把结果写回输出文件中对应列。  
   **`--arm both`** 时：每帧对**同一行**轨迹设好全身初值后，**先完成左臂整条 IK，再完成右臂 IK**（右臂计算时 `qpos` 中已含左臂更新）。

内置 **手指列名别名**（`CASBOT_FINGER_DATA_TO_MJCF`）：例如数据里 `left_thumb_metacarpal_joint` 会映射到 MJCF 的 `left_thumb_mcp_roll_joint`，与 `DZQ_*.data` 一类导出格式兼容。

### 1.2 模型格式

- **`--robot-model`** 须为 **`.urdf`**（Pinocchio）。FK/IK 已完全切换到 Pinocchio，不再接受 `.xml`/`.mjcf`。

### 1.3 主要参数

| 参数 | 说明 |
|------|------|
| `--robot-model` | URDF 路径（必填） |
| `--data` | 输入轨迹（必填） |
| `--output` | 输出路径，默认 `data_transform.csv` |
| `--arm` | `left`、`right` 或 **`both`**（双臂各用一组 H） |
| `--offset-left` | 可选，左臂 `x,y,z,rx,ry,rz` 逗号分隔（单位同下）；首项为负须 `--offset-left=-1,0,...`，见 **§5.1** |
| `--offset-right` | 可选，右臂同上；首项为负须 `--offset-right=-50,0,...`，见 **§5.1** |
| `--x --y --z` | 单臂时或 **`both` 且未写 offset 时**：双臂共用该 6 元组；单臂时也可用 `--offset-left`/`--offset-right` 覆盖 |
| `--rx --ry --rz` | 欧拉角分量，单位由 `--angle-unit` 决定 |
| `--length-unit` | `mm`（默认）或 `m` |
| `--angle-unit` | `rad`（默认）或 `deg` |
| `--euler-seq` | 传给 `scipy` 的欧拉序列，默认 `XYZ`（外旋固定轴 X→Y→Z） |
| `--compose` | `left` 或 `right`，见上文 |
| `--ee-body` | 单臂时末端 body；**`both` 时**请用 `--ee-body-left` / `--ee-body-right` |
| `--ee-body-left` / `--ee-body-right` | 双臂末端 body，可选 |
| `--arm-joints` | 单臂 7 关节；**`both` 时**建议 `--arm-joints-left` / `--arm-joints-right` |
| `--arm-joints-left` / `--arm-joints-right` | 双臂关节名列表，可选 |
| `--ik-joints` | 单臂 IK 链；**`both` 时**可用 `--ik-joints-left` / `--ik-joints-right` |
| `--ik-joints-left` / `--ik-joints-right` | 可选 |
| `--write-joints` | 单臂写回列；**`both` 时**建议 `--write-joints-left` / `--write-joints-right` |
| `--write-joints-left` / `--write-joints-right` | 可选 |
| `--urdf` | 可选，关节名对照 |
| `--ik-iters` | IK 最大迭代步数，默认 80 |
| `--ik-tol` | 位姿误差阈值，默认 `1e-4` |
| `--ik-damping` | 阻尼最小二乘系数，默认 `1e-3` |
| `--ik-max-step` | 每步关节增量 L2 上限（弧度），默认 `0.15` |

**偏移约定（`arm_transform_common.collect_arm_passes`）**：

- **`--arm left`**：优先 **`--offset-left`**，否则用 **`--x … --rz`**。
- **`--arm right`**：优先 **`--offset-right`**，否则用 **`--x … --rz`**。
- **`--arm both`**：若 **`--offset-left` 与 `--offset-right` 均未设置**，则左右共用 **`--x … --rz`**；若只设一侧 offset，另一侧 **`H` 为单位变换**（该侧关节不写偏移）。两侧均设 offset 时各自独立。

若某条手臂的 **IK 链关节**（如含 **`waist_yaw_joint`**）多于 **写回列**，程序会在标准错误输出提示：仅用输出列做 FK 可能与 `H ⊗ FK` 不完全一致。

### 1.4 示例

左臂、世界系下沿 **Y 平移 10 mm**、旋转为 0（平移用毫米，角用度）：

```bash
python3 arm_trajectory_transform.py \
  --robot-model urdf/CASBOT02_ENCOS_7dof_shell_20251015_P1L.urdf \
  --data data/DZQ_PianYi_100HZ.data \
  --output data/DZQ_PianYi_100HZ_transform.data \
  --arm left \
  --x 0 --y 10 --z 0 --rx 0 --ry 0 --rz 0 \
  --length-unit mm --angle-unit deg
```

IK 难以收敛时，可增大 `--ik-iters` 或调整 `--ik-damping`；必要时在 **`--ik-joints`** 中加入 `waist_yaw_joint`，并用 **`--write-joints`** 决定是否把腰关节一并写入输出。

**双臂示例**（左臂 +Y 1 cm，右臂 −Y 1 cm，单位米）：

```bash
python3 arm_trajectory_transform.py \
  --robot-model urdf/CASBOT02_ENCOS_7dof_shell_20251015_P1L.urdf \
  --data data/DZQ_PianYi_100HZ.data \
  --output data/dual_arm.data \
  --arm both \
  --offset-left "0,0.01,0,0,0,0" \
  --offset-right "0,-0.01,0,0,0,0"
```

每帧内顺序为：**先左臂 IK，再右臂 IK**（均在同一行轨迹设初值后）。**`--arm both`** 时建议为左右分别指定 **`--write-joints-left` / `--write-joints-right`**，避免误用全局 **`--write-joints`**（仅 7 个左或右名称时另一侧列不会更新）。

## 二、`arm_trajectory_transform_jacobian.py`

### 2.1 功能概要

与第一节相同的：读轨迹 → 按行写入 `qpos` → **FK** 得 `T_track_end` → 构造 **`T_end_transform = H ⊗ T_track_end`**（`H` 与 `--compose` 规则相同）。

区别在**求新关节角**的方式：

1. **目标位姿** `T_end_transform` 在每帧只算一次（由**原始**该帧 `track_joint` 对应的 FK 决定），整帧内不变。
2. **默认 `--linear-steps=2`（两步）**  
   - **第一步**：在当前 `q`（原始轨迹位形）下算 `J(q)`、位姿误差 `e`，阻尼最小二乘得 `Δq₁`，叠加并限位；  
   - **第二步**：在 **`q + Δq₁`** 上重新 FK、算 `J(q+Δq₁)`、再求 `Δq₂` 并叠加。  
3. 写回对应 **`--write-joints*`** 列为最终 `q`（默认每侧仍为该侧手臂 7 关节）。**`--arm both`** 时顺序与第一节相同：**先左后右**，每条手臂在**当前** `q` 上算雅可比。

每步公式与迭代 IK 中的**单步**相同（\(e \approx J\,\Delta q\)），但**不**循环到 `--ik-tol`；步数由 **`--linear-steps`** 固定（设为 `1` 即单步）。

### 2.2 主要参数

除下列以外，**`--robot-model`、`--data`、`--output`、`--arm`（含 `both`）、`--offset-left`/`--offset-right`、平移/角度单位、`--euler-seq`、`--compose`、双臂 `--ee-body-*` / `--arm-joints-*` / `--write-joints-*`、`--urdf`** 与第一节含义一致（雅可比链对应 **`--jac-joints*`**，见下；输出默认 `data_transform_jacobian.csv`）。

| 参数 | 说明 |
|------|------|
| `--jac-joints` | 单臂雅可比关节链；**`both` 时**用 `--jac-joints-left` / `--jac-joints-right` |
| `--jac-joints-left` / `--jac-joints-right` | 可选 |
| `--linear-steps` | 雅可比增量次数，**默认 `2`**；每步都在**当前** `q` 上重算 `J` |
| `--damping` | 阻尼最小二乘 λ，默认 `1e-3` |
| `--max-delta-norm` | 单次 `Δq` 的 L2 上限（弧度），默认 `0.15` |

若 **`--jac-joints*`** 比 **`--write-joints*`** 多（例如用了腰但未写回），同样会打印与 FK 不一致的提示。

### 2.3 示例

与第一节相同变换、改用雅可比两步脚本（左臂单臂）：

```bash
python3 arm_trajectory_transform_jacobian.py \
  --robot-model urdf/CASBOT02_ENCOS_7dof_shell_20251015_P1L.urdf \
  --data data/DZQ_PianYi_100HZ.data \
  --output data/DZQ_PianYi_100HZ_transform_jacobian.data \
  --arm left \
  --x 0 --y 10 --z 0 --rx 0 --ry 0 --rz 0 \
  --length-unit mm --angle-unit deg
```

**双臂**（与第一节 IK 双臂示例相同的 offset）：

```bash
python3 arm_trajectory_transform_jacobian.py \
  --robot-model urdf/CASBOT02_ENCOS_7dof_shell_20251015_P1L.urdf \
  --data data/DZQ_PianYi_100HZ.data \
  --output data/dual_arm_jacobian.data \
  --arm both \
  --offset-left "0,0.01,0,0,0,0" \
  --offset-right "0,-0.01,0,0,0,0"
```

只要 **一步** 时：`--linear-steps 1`。需要更多线性化迭代：`--linear-steps 3` 等。

## 三、`mujoco_trajectory_sim.py`（仅 x86_64）

> **注意**：本脚本仅在 x86_64 Docker 环境中可用。在 Orin / aarch64 上运行会直接报错退出。

### 3.1 功能概要

- 加载 **MJCF/XML**（`--robot-model`）。
- 读取与变换脚本相同格式的 **`.data`**（首行列名，逗号分隔）。
- 默认：每帧将能映射到模型的关节值写入 `qpos`，调用 **`mj_forward`**；在被动 Viewer 中按 **`--hz`** 定时刷新。
- 若设置 **`--tf-arm`**：每帧在写入轨迹后，对左/右/双臂施加与 **`arm_trajectory_transform.py` 相同的 SE(3)+迭代 IK**（Pinocchio；参数前缀 **`--tf-`**），再 **`mj_forward`**。适合在未离线生成新 `.data` 时预览左右偏移。  
  **顺序**：与离线脚本一致，**先左后右**。实时模式**未暴露** `--ik-joints-left` 等细调，两臂均用默认 **7 自由度** 手臂链；需要腰转等请先用离线 **`arm_trajectory_transform.py`** 生成新 `.data`。
- 表中无法映射的列（如 `Base_*`、速度列等）自动跳过；手指列名使用与 **`arm_trajectory_transform*.py`** 相同的别名表。

说明：当前 CASBOT MJCF 以 **motor** 为主，本脚本是**直接设关节位置**的回放，不是完整力矩/接触动力学仿真。

### 3.2 主要参数

| 参数 | 说明 |
|------|------|
| `--robot-model` | MJCF/XML 路径（必填） |
| `--data` | 轨迹 `.data`（必填） |
| `--hz` | 回放帧率，默认 `100` |
| `--speed` | 相对实时倍速，默认 `1.0` |
| `--loop` | 播放结束后从头循环 |
| `--no-viewer` | 不打开窗口，只快速跑完轨迹（便于校验） |
| `--max-frames` | 最多播放帧数，`0` 表示全部 |
| `--tf-arm` | 可选，`left` / `right` / `both`，启用实时 IK 偏移 |
| `--tf-offset-left` / `--tf-offset-right` | 与变换脚本相同的 6 元组字符串；首项为负须 `--tf-offset-right=-50,...`，见 **§5.1** |
| `--tf-x` … `--tf-rz` | 未指定 offset 时的共用或单臂数值（单位见 `--tf-length-unit` / `--tf-angle-unit`） |
| `--tf-length-unit` / `--tf-angle-unit` / `--tf-euler-seq` / `--tf-compose` | 与变换脚本一致；平移默认 `mm`（`--tf-length-unit m` 为米） |
| `--tf-ik-iters` | 默认 `60`（略低于离线脚本默认 `80`，可按需加大） |
| `--tf-ik-tol` / `--tf-ik-damping` / `--tf-ik-max-step` | 与离线 `--ik-*` 含义一致 |

启用 **`--tf-arm`** 后每帧会多次 **`mj_forward` + IK**，CPU 占用明显高于纯回放。

### 3.3 示例

可视化播放（需图形环境，仅 x86）：

```bash
python3 mujoco_trajectory_sim.py \
  --robot-model urdf/xml/CASBOT02_ENCOS_7dof_shell_20251015_P1L.xml \
  --data data/DZQ_PianYi_100HZ.data \
  --hz 100
```

播放变换后的轨迹、无可视、只跑前 200 帧：

```bash
python3 mujoco_trajectory_sim.py \
  --robot-model urdf/xml/CASBOT02_ENCOS_7dof_shell_20251015_P1L.xml \
  --data data/DZQ_PianYi_100HZ_transform.data \
  --no-viewer --max-frames 200
```

**不导出文件、仅在回放时双臂在线偏移**（单位：米 / 弧度，可与离线示例对照）：

```bash
python3 mujoco_trajectory_sim.py \
  --robot-model urdf/xml/CASBOT02_ENCOS_7dof_shell_20251015_P1L.xml \
  --data data/DZQ_PianYi_100HZ.data \
  --hz 100 \
  --tf-arm both \
  --tf-offset-left "0,0.01,0,0,0,0" \
  --tf-offset-right "0,-0.01,0,0,0,0"
```

## 四、典型工作流

1. 用 **`arm_trajectory_transform.py`** 或 **`arm_trajectory_transform_jacobian.py`** 对原始 `*.data` 施加末端变换 **H**（支持 **`--arm both`** 与 **`--offset-left` / `--offset-right`**），得到不同的 `*_transform*.data`。
2. **x86 环境**：用 **`mujoco_trajectory_sim.py`** 分别加载原始与变换后轨迹，对比动画或做快速无头校验；或临时加 **`--tf-arm`** 试左右偏移而无需先写新文件。
3. **aarch64 / Orin**：直接将变换后的 `*.data` 部署到真机上测试（不支持 MuJoCo 仿真）。

## 五、常见问题

- **变换脚本只认 URDF**：`arm_trajectory_transform*.py` 通过 Pinocchio 读 URDF 模型，不再接受 `.xml`/`.mjcf`。
- **仿真脚本只认 MJCF/XML**：`mujoco_trajectory_sim.py`（仅 x86）通过 MuJoCo 读 MJCF/XML。
- **轨迹与模型关节名不一致**：手臂、腿、头、腰等名称需与模型中 `<joint name="...">` 一致；手指请依赖内置别名或自行改 `CASBOT_FINGER_DATA_TO_MJCF`（两个 transform 脚本内定义一致）。
- **Viewer 黑屏/无法启动**：在无图形环境（如部分远程服务器）使用 **`--no-viewer`**。
- **aarch64 运行 mujoco_trajectory_sim 报错**：这是预期行为，MuJoCo 仿真仅支持 x86_64。

### 5.1 命令行：`--offset-*` / `--tf-offset-*` 首项为负数

`argparse` 会把**下一个**以 `-` 开头的单词当成**新的可选参数**。因此下面写法会报错（如 `argument --offset-right: expected one argument`），**即使**在 shell 里给整串加了引号：

```bash
# 错误示例
--offset-right "-50,0,0,0,0,0"
```

**正确**：用**等号**把 6 元组粘在选项名上，整段只算一个参数：

```bash
--offset-right=-50,0,0,0,0,0
--offset-left=-1,0,0,0,0,0
# 仿真里在线偏移同理：
--tf-offset-right=-50,0,0,0,0,0
```

首项**非负**时，`--offset-right 0,50,0,0,0,0` 与 `--offset-right=0,50,0,0,0,0` 一般都可以。

对 **`--x --y --z`** 等单独数值选项，若传入负数，同样建议 **`--y=-10`** 这种等号形式，避免被误解析。

### 5.2 轨迹文件（CSV / `.data`）里的负数

数据表单元格中的负关节角、负平移等由 **`read_trajectory_table`** 按 `float()` 解析，**与命令行规则无关**，按正常 CSV 写负数即可（如 `-0.185207`、`,-5,`）。

更详细的实现说明见各脚本及 **`arm_transform_common.py`** 顶部的模块文档字符串。
